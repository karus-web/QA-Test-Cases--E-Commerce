# QA Test Cases – E-commerce Application

I created this project as part of my QA learning to practice writing manual test cases based on real-world scenarios.

The focus of this project is to understand how users interact with an e-commerce application and how to test those workflows step by step.

---

## 🚀 What I Covered

In this project, I worked on testing the following features:

- User Signup  
- Login functionality  
- Order placement  
- Order details  
- Payment process  
- Payment cancellation  

---

## 🧪 How I Tested

While creating these test cases, I focused on:

- Valid user actions (positive scenarios)  
- Invalid inputs and error handling (negative scenarios)  
- Field validations and edge cases  
- End-to-end user flows (from signup to payment)  

---

## 🔗 Workflow I Followed

1. User signs up → account is created  
2. User logs in → authentication is verified  
3. User places an order → order is generated  
4. Order details are checked  
5. Payment is processed  
6. Payment cancellation is tested
